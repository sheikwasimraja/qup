Quran Progress Tracker Guide

This guide explains how to launch, use, and keep your own progress in the GitHub Pages version of the tracker.

Open Tracker Open Summary Open Trends Open Settings

Why Keep Going

Learning the Quran is a journey of patience, consistency, and barakah. Allah says, "And We have certainly made the Quran easy to remember. So is there anyone who will be mindful?" (Quran 54:17). The Prophet (peace be upon him) said, "The best among you are those who learn the Quran and teach it." (Sahih al-Bukhari 5027). Even small daily progress matters.

1. What This Version Uses

  • quran-progress-tracker.html is the main tracker.
  • quran-progress-summary.html shows progress overview charts.
  • quran-progress-trends.html shows daily and monthly trends.
  • quran-reference-settings.html controls reference links and visible columns.
  • Your verse progress is saved in your browser on your own device.

2. How To Launch

  1. Upload this folder to a GitHub repository.
  2. Enable GitHub Pages for that repository.
  3. Open the published site URL.
  4. Use the tracker directly in the browser. No Node.js or local server is required.

3. Daily Use

  1. Search, filter, or use the header filters under each column.
  2. Update the LEARNT and MEMORISE dropdowns row by row.
  3. When you change LEARNT to Yes, the tracker fills DATE LEARNT automatically.
  4. When you change MEMORISE to Yes, the tracker fills DATE MEMORISED automatically.
  5. Click Save Progress to save your current work in the browser.

4. Settings

  1. Open Reference Settings from the tracker.
  2. Choose which reference sources should open.
  3. Choose which columns should be visible.
  4. Save the settings and refresh the tracker if it is already open.

Features Available

  • Main table with search, paging, and column-wise filters.
  • Random Review Verse to pick a random verse not yet marked as learnt.
  • Today's Progress to show rows learnt today.
  • Updated verses to show only the rows changed in your current session.
  • Open all references to open the selected reference websites for a verse.
  • Close Reference Tabs to close reference tabs opened by the tracker when the browser allows it.
  • Summary and trend pages that reflect your current browser-saved progress.

How Saving Works

Your progress is saved in your browser on your own device. If another person opens the same site in their own browser, they will keep their own separate progress. This version does not need a server, but clearing browser storage will remove saved progress for that browser.

Sharing With Someone Else

  1. Share the GitHub repository or GitHub Pages link.
  2. Each person can use the same site independently.
  3. Each browser keeps its own saved progress.
  4. No one overwrites another person's learning progress unless they share the same browser storage.

Troubleshooting

  • If the tracker looks old, refresh the page once.
  • If charts do not match the tracker, save your progress and refresh the chart page.
  • If settings do not appear, save them and then refresh the tracker.
  • If a reference tab does not close, that website may block script-based closing after it opens.
  • If browser data is cleared, the tracker returns to its original seeded data.
